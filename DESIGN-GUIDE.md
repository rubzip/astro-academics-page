Act as an expert UI/UX Frontend Developer. 
Strictly follow these UI/UX design guidelines:

1. GLOBAL VIBE & LAYOUT:
- Ultra-minimalist, content-first, and developer-focused.
- No clutter, no unnecessary visual noise. 
- Use a single, centered main container with a strict max-width (e.g., max-w-3xl or 800px).
- Generous vertical whitespace between sections for readability.

2. COLOR PALETTE:
- Monochromatic and high-contrast base. 
- Light Mode: Off-white/very light gray background (e.g., #fafafa or #fefefe) with near-black text (#222).
- Dark Mode: Very dark charcoal/slate background (e.g., #212529) with off-white text (#fbfbfb).
- Accent Color: Use ONE single accent color very sparingly (e.g., a muted skin-color/orange or a subtle link color) only for hover states, active states, or subtle highlights.

3. TYPOGRAPHY:
- System fonts or clean sans-serif (like Inter or Roboto) for all body text and headings.
- Monospace font (like JetBrains Mono or Fira Code) for code blocks, tags, or small technical metadata.
- Focus heavily on text hierarchy: large bold titles, clear subheadings, and highly readable line-heights (leading-relaxed) for paragraphs.

4. UI ELEMENTS & COMPONENTS:
- Flat design. STRICTLY NO drop shadows, NO glassmorphism, and NO heavy gradients.
- If separation is needed, use subtle 1px solid borders or very faint background color variations.
- Buttons and Links: Keep them simple. Links should have a hover effect (e.g., underline offset, or SVG icon slightly moving right).
- Tags: Small, pill-shaped or slightly rounded borders with a monospace font.
- Icons: Use minimal, outline-style SVG icons (like Lucide or Tabler Icons), sized appropriately so they don't overpower the text.

5. ACCESSIBILITY (a11y):
- Contrast ratios must pass WCAG AAA.
- Include clear focus rings for keyboard navigation (outline-dashed, offset).
